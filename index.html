News

June 2021: I recently stumbled across a YoutTube video by Numberphile, which discussed finding integer solutions for the expression x^3+y^3-z^3=k, where k=42. The solution found has x, y and z values with 16 digits. Currently the only known way to find solutions to equations like this is to search all the possibilities looking for matches, and testing 10^[48] values isn't currently feasible. The search can be reduced to order n^2 (a still impractical 10^[32] values) by noticing that for small 'k' and large 'x', 'y' and 'z', z~=−(x^3+y^3)^[1/3]. What made the video interesting is that they had an algorithm that they claimed was order 'n', provided you focused on a specific 'z'. The video didn't go into details of how this was implemented so I thought I'd have a go myself. The results can be found on my Searching for Cube Sums page.

A few things to note:

  • My page only describes how the algorithm works, in includes demonstrations written in Javascript, which will allow you to search for some numbers, k=72 and k=83 work quite well, but as the algorithm involves calculating z^3 you won't be able to go beyond z=2^[18] (the cube root of the maximum safe integer).
  • I could rewrite it using unlimited length integers (I have a Javascript class that does that), but then you have to consider that the Numberphile people ran their calculation for days on a rack of computers with 512 cores, probably using something that runs faster than Javascript in a browser.
  • My implementation involves finding the prime factors of z^3-k. This isn't linear in 'n' so the whole calculation isn't quite of order 'n', The video didn't discus this, maybe they have a clever way around it.
  • As 42 modulo 9 is 6, the solution can only include 'x', 'y' and 'z' values that are '2' modulo '3', eliminating 2/3 of the possibilities for 'z'. This is explained on the page.
